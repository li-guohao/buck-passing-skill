# 何时不该使用甩锅-skill / When NOT to Use Buck-Passing Skill

## 🚫 绝对不要使用的情况 / Absolute Prohibitions

### 1. 明显是自己的错误 / When It's Clearly Your Mistake

**场景示例**:
```
❌ 你忘记了提交代码，导致上线延期
❌ 你没有测试就发布了功能，导致线上故障
❌ 你漏看了邮件，错过了重要会议
```

**为什么不该甩锅**:
- 会严重损害你的信誉
- 同事和上级会看穿
- 失去成长机会
- 破坏团队信任

**正确做法**:
```
✅ 立即承认错误
✅ 提出补救方案
✅ 制定预防措施
✅ 使用[背锅-skill](https://github.com/li-guohao/back-taking-skill)主动担当
```

---

### 2. 涉及安全和合规问题 / Safety and Compliance Issues

**场景示例**:
```
❌ 数据泄露事故
❌ 安全漏洞被利用
❌ 违反法律法规
❌ 侵犯用户隐私
```

**为什么不该甩锅**:
- 可能涉及法律责任
- 公司需要真实原因来修复问题
- 甩锅可能导致更严重的后果
- 影响公司声誉和客户信任

**正确做法**:
```
✅ 立即报告真实情况
✅ 配合调查
✅ 参与制定修复方案
✅ 承担相应责任
```

---

### 3. 团队已经达成共识 / Team Has Reached Consensus

**场景示例**:
```
❌ 团队已经认定是A的责任，你为了自保而甩锅给B
❌ 大家已经接受现状，你重新挑起责任争议
❌ 问题已经解决，你为了洗白自己而翻旧账
```

**为什么不该甩锅**:
- 破坏团队和谐
- 显得小肚鸡肠
- 浪费大家时间
- 影响你的职业形象

**正确做法**:
```
✅ 接受团队决定
✅ 专注于未来改进
✅ 私下澄清（如果必要）
✅ 向前看，不纠结过去
```

---

## ⚠️ 谨慎使用的情况 / Use with Caution

### 1. 上级直接问责 / When Superior Directly Questions

**风险**: 容易被视为推卸责任、不尊重上级

**建议**:
- 先承认自己的部分责任
- 客观陈述事实，不带情绪
- 提供解决方案，而非借口
- 使用温和型话术（Level 1-2）

### 2. 跨部门冲突 / Cross-department Conflicts

**风险**: 可能升级为部门间矛盾

**建议**:
- 强调共同目标
- 指向流程问题而非个人
- 寻求第三方协调
- 避免公开指责

### 3. 涉及人事关系 / Involving Personal Relationships

**风险**: 可能伤害人际关系

**建议**:
- 私下沟通，不在公开场合
- 给对方留面子
- 寻求双赢解决方案
- 必要时请HR介入

---

## ✅ 适合使用的情况 / Appropriate Usage

### 1. 责任确实不明确 / Responsibility Is Truly Unclear

**场景**:
```
✅ 多个部门协作，边界模糊
✅ 历史遗留问题，无法追溯
✅ 系统性问题，非个人能控制
```

### 2. 被冤枉或误解 / Being Wronged or Misunderstood

**场景**:
```
✅ 决策是上级做的，但责任推给你
✅ 其他部门延误导致你这边延期
✅ 需求变更频繁导致工作量增加
```

### 3. 保护团队利益 / Protecting Team Interests

**场景**:
```
✅ 团队被不合理指责
✅ 其他部门甩锅给你的下属
✅ 需要为团队争取合理权益
```

### 4. 促进问题解决 / Facilitating Problem Solving

**场景**:
```
✅ 指出系统性问题，推动改进
✅ 澄清责任边界，避免未来争议
✅ 客观分析，帮助找到root cause
```

## 🎯 决策流程 / Decision Flow

```
问题发生
    |
    ↓
是否明显是我的错? 
    ├── 是 → 使用[背锅-skill]主动担当
    |
    └── 否 → 继续判断
                |
                ↓
        责任是否清楚?
            ├── 是，责任在我 → 背锅
            ├── 是，责任在他 → 甩锅（谨慎）
            └── 否，不清楚 → 分析后再决定
                |
                ↓
        是否涉及安全/合规?
            ├── 是 → 绝不甩锅，如实报告
            └── 否 → 继续判断
                |
                ↓
        团队是否已达成共识?
            ├── 是 → 接受共识，向前看
            └── 否 → 可以使用甩锅-skill
                |
                ↓
        选择合适等级和话术
```

## 💡 黄金法则 / Golden Rules

1. **诚实第一** / Honesty First
   - 不要歪曲事实
   - Don't distort facts

2. **对事不对人** / Focus on Issues, Not People
   - 指向问题本身，而非个人
   - Point to the problem itself, not individuals

3. **建设性导向** / Constructive Orientation
   - 目标是解决问题，而非逃避
   - Goal is to solve problems, not evade them

4. **留有余地** / Leave Room
   - 给对方台阶下
   - Give the other party a way out

5. **长期视角** / Long-term Perspective
   - 考虑对职业发展的长期影响
   - Consider long-term impact on career development
